1. 슬라이딩 윈도우
<대상>
- 1차원 배열 / 문자열

<사용처>
- 연속된 구간(subarray / substring)
- 구간의 합, 길이, 개수, 최대/최소

<핵심 개념>
- left, right 모두 0부터 시작
- 조건 확인 시 right 확장
- 조건 위반 시 left 이동하여 창 축소

///

2. 투 포인터
<요소>
정렬된 리스트

<사용처>
- 두 값의 합
- 특정 조건을 만족하는 쌍(pair)
- 조합 탐색

<접근>
[보통의 경우]
left = 0, right = n-1
while left < right:
    조건에 따라 left++, right--

/// 

3. 미들 인 더 미들
<요소>
리스트

<사용처>
요소의 개수가 클 때, 냅색문제

<접근>
- 배열을 반으로 나눔
- 한쪽 고정 + 다른 쪽 이분 탐색
- 다른쪽의 경우: start와 end를 통해 mid를 정하고, left_list[mid]를 사용하여 조건 탐색
